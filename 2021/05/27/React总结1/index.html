React总结（一）：React组件基础

1. React事件机制

React中JSX上写的事件并不会被绑定到对应的真实DOM中，而是统一的绑定到document上。
事件发生后，通过事件冒泡冒泡到document处，React将事件内容封装并且交给真正的处理函数运行处理。
React上的事件采用的是合成事件SyntheticEvent，而不是原生浏览器事件。

1.1 合成事件

合成事件是React模拟原生DOM事件所有能力的一个事件对象，优点如下：

  • 兼容所有浏览器，具有更好的跨平台能力
  • 将事件统一存放在一个数组也就是事件池中，避免频繁的新增与删除（垃圾回收）
  • 方便react统一管理和事务机制
  • 对于原生浏览器事件，浏览器会给监听器创建一个事件对象，如果有很多个事件监听就需要分配很多的事件对象，会造成高额内存分配问题。对于合成事件，有一个专门的事件池来管理这些事件的合成与销毁，事件需要被使用时，就从事件池中复用对象，事件回调结束后，就销毁事件对象上的属性，以便于下次复用事件对象。

合成事件与HTML原生事件区别：

  • 事件名称命名：原生事件为全小写，react事件采用小驼峰
  • 事件函数处理语法：原生事件为字符串，react事件为函数
  • react事件必须采用even.preventDefault()来阻止默认行为。
    不能采用event.stopPropagation或者是return false。
  • 事件执行顺序：原生事件先执行，合成事件后执行，合成事件会冒泡绑定到document上，所以尽量避免原生时间与合成事件混用，如果要阻止原生事件冒泡，可能会导致合成事件不执行，因为需要冒泡到document上合成事件才会执行。

1.2 React事件代理

React基于Vitrtual DOM实现了一个合成事件层SyntheticEvent层，定义的事件处理器会接收到一个合成事件对象的实例，它符合W3C标准，并且和原生浏览器事件具有相同的接口，支持冒泡机制，所有事件都是绑定在最外层上的。

在React底层，主要对合成事件做了两件事：

  • 事件委派：React会把所有的事件绑定到最外层，使用统一的事件监听器，这个事件监听器上维持了一个映射来保存所有组件内部事件监听和处理函数。
  • 自动绑定：React组件中，每个方法的上下文都会指向该组件的实例，即自动绑定this为当前组件。

2.React Hooks 解决代码复用

三种解决React代码复用的方式：高阶组件HOC，render props，hooks

2.1 React高阶组件HOC 是一个函数

高阶组件HOC是React中用于复用组件逻辑的一种高级技巧。HOC本身并不是React API的一部分，它是一种基于React的组合特性而形成的设计模式。HOC是一种组件的设计模式，HOC接受一个组件和额外的参数，返回一个新的组件。HOC是一个纯函数，没有副作用

  • 优点：HOC函数包裹组件进行复用，不影响被包裹组件的内部逻辑。
  • 缺点：HOC传递给被包裹组件的props容易和被包裹后的组件重名，进而被覆盖

2.2 Render props

render prop指在React组件之间使用一个值为函数的prop共享代码的简单技术。

  • 优点：数据共享、代码复用，将组件内的state作为props传递给调用者，将渲染逻辑交给调用者
  • 缺点：无法在return语句外访问数据，嵌套写法不够优雅

2.3 Hooks

React Hooks是React 16.8新出的特性。可以在不使用class组件的情况下使用state以及其他React的特性。通过自定义hook，可以复用代码逻辑。常用的有useState,useEffect等

  • 优点：

    • 使用直观
    • 解决了hoc高阶函数props可能会被覆盖的问题
    • 解决了render props因共享数据出现嵌套地狱的问题
    • 解决了render props能在return之外使用数据的问题

    总结：React Hooks yyds！！！！！！！！

3. React Fiber协程

Fiber成为协程或者是纤程。他和线程并不一样，线程拥有并行能力，协程本身没有并发或者并行能力，需要配合线程才能拥有并行能力。协程是一种控制流程的让出机制，在适当的时候让出CPU的执行权，让CPU能在这段时间去执行其他的操作。渲染的过程是可以被中断的，可以将控制权交给浏览器，让位给优先级高的让你无，浏览器空闲后再恢复渲染。

场景：React V15在渲染的时候，会递归对比VirtualDOM树，找出需要变动的节点，然后同步一口气更新他们。这个过程期间，React会长期占用浏览器资源，这就会导致用户出发浏览器事件得不到相应，并且会导致掉帧，用户体验十分卡顿。

为了解决这个问题，给用户一种很快的假象，协程应用而生。协程不让一个任务长期霸占资源。可以将浏览器的渲染、布局、绘制、资源加载、事件响应、脚本执行等都视作浏览器的“进程”，利用协程去调度策略合理的分配CPU资源，提高浏览器的响应速率，同时兼顾任务执行效率。

React通过Fiber框架，让执行过程变成可中断的。

  • 适度让出CPU执行权，提高浏览器的响应速率，让浏览器及时响应用户的交互
  • 分批延时对DOM进行操作，可以避免一次性操纵大量DOM节点，优化用户体验
  • 给浏览器一些喘息机会，它会对代码进行编译优化JIT和热代码优化，或者对reflow进行修正

4. 函数组件与类组件

  • 相同点

    函数组件和类组件都是组件。组件是React可以复用的最小代码片段/最小编码单元，他们都会返回要在页面中渲染的React元素。作为最小编码单元，在使用方式和最终呈现效果上都是完全一致的。类组件和函数组件可以相互改写（有状态时只能用类组件，或者用函数组件hooks）

  • 不同点

    • 类组件是面向对象编程，主打继承、生命周期等核心概念

      函数组件是函数式编程，主打immutable（不变）、没有副作用、引用透明等特点

    • 在React Hooks之前，如需生命周期、继承，必须使用类组件。有了React Hooks之后，生命周期概念淡出，函数组件可以完全取代类组件。并且继承并不是组件设计的最佳设计模式，官方推崇“组合优于继承”。总之，React Hooks函数组件完胜类组件。

    • 性能上，类组件依靠shouleComponentUpdate阻断渲染来提升性能，而函数组件依靠React.memo缓存渲染结果提升性能。

    • 函数组件少了类组件中繁杂的this的概念，减少学习成本，更轻量渐变，而且在Hooks的基础上提供了更细粒度的逻辑组织与复用。

5. React Class 组件 构造函数

类组件构造函数有两个主要目的：

  • 通过将对象分配给this.state来初始化本地状态
  • 将事件处理程序方法绑定到实例上

构造函数用来新建父类的this对象。
子类必须在constructor中调用super方法，否则子类会得不到this对象
子类没有自己的this对象，而是继承父类的this的对象，然后对他进行加工。
constructor()内部如果要使用this.props就将props传入，否则不用
js中bind每次都会返回一个新的函数，为了性能考虑，尽量都在constructor中bind事件
