CSS拾遗（二）display属性

发表于2021-05-27|更新于2021-05-30|CSS
|阅读量:

1 display属性

display属性：用来控制一个元素及其子元素的格式化上下文

display属性从两个维度来描述

外部显示类型：确定一个元素在普通文档流中的表现，比如式块级元素还是内联元素等等

内部显示类型：定义了元素内子元素的布局方式，改变其子元素的格式化上下文

display属性值有32中，分为以下大类：

  • 外部值
  • 内部值
  • 列表值
  • 属性值
  • 显示值
  • 混合值
  • 全局值

1.1 外部值

定义元素的外部表现

属性值 双值 描述
block block flow 外部表现为块级元素，不影响内部
inline inline flow 外部表现为内联元素，不影响内部
run-in 生成一个插入框，是一个特殊的内联，试图合并到后续块容器中

1.2 内部值

定义元素的子孙的表现也就是内部表现

属性值 双值 描述
flow 如果它的外部显示类型是内联inline或run-in，并且它正在参与一个格式化上下文（inline/block），那么它将生成一个内联元素。否则，他将生成一个块级元素。
flow-root block flow-root 元素生成一个块级元素，并使用flow布局其内容。它总是为其内容建立一个新的块格式上下文。
table 元素会作为块级表格来显示，表格前后带有换行符。
flex block flex 元素会生成一个弹性容器，并且生成一个弹性格式化上下文
grid block grid 元素会生成一个网格容器，并且生成一个网格格式化上下文
ruby 注记，元素生成一个ruby容器盒并建立一个ruby格式化上下文。还将其内容集成到其父格式化上下文中(如果是内联的)，或者生成一个适当的外部显示类型的包装器框(如果不是)。

1.3 列表值

将内容以列表的形式呈现

如果没有指定内部显示类型值，则主体框的内部显示类型默认为流。

如果没有指定外部显示类型值，则主体框的外部显示类型默认为块。

1.4 属性值

属性值一般是附属于主值的,为table-xx/ruby-xx，为table或者ruby布局模型的内部结构布局，用于定义他们子孙的布局

table-row-group*,** table-header-group**,** table-footer-group**,** table-row**,** table-cell**,** table-column-group**,** table-column* 它们都是内部表格元素，它生成适当的内部表格框，在表格格式化上下文中。

table-caption它生成一个表格标题框，是一个特殊块级容器框

ruby-base*,** ruby-text**,** ruby-base-container**,** ruby-text-container*它们是一个内部ruby元素，它生成适当的内部ruby框，该框参与ruby格式化上下文

1.5 显示值（盒子值）

控制一个元素是否生成box（框），具有这两个值中的任何一个的元素都没有内部或外部显示类型，因为它们根本不生成任何box（框）

display：content

元素本身不生成任何box，但它的子元素和伪元素仍然生成box，文本照常运行。为了box生成和布局，元素必须被视为已被其取代元素树中的内容。由于只影响盒子树，因此基于文档树的任何语义，如选择器匹配、事件处理和属性继承，都不受影响

display： none

元素及其后代不生成盒子或运行文本

1.6 混合值

属性值 双值 描述
inline-block inline flow-root 行内块元素（CSS2.1 新增的值）
inline-table inline table 在行内进行表格布局
inline-flex inline flex 在行内进行弹性布局
inline-grid inline grid 在行内进行网格布局

1.7 全局值

这些值不是 display属性的专利，几乎其它任意属性都可以用，列在这里凑个数。

display: inherit

继承父元素的 display属性。

display: initial

不管父元素怎么设定，恢复到浏览器最初始时的 display属性。

display: unset

unset混合了 inherit和 initial。如果父元素设值了，就用父元素的设定，如果父元素没设值，就用浏览器的缺省设定。

2 元素隐藏的几种方式

2.1 display:none

元素在页面上彻底消失，元素本来占有的空间会被其他页面元素占有。

也就是说，会导致浏览器的重排和重绘。

2.2 visibility:hidden

元素在页面看不到，但是它占据的空间仍会保留。

也就是说，会导致浏览器的重绘，不会重排

2.3 opacity:0

设置元素透明度为0后，元素完全透明，看起来好像消失了，其实还在

这种情况下，不会改变页面原有的布局，元素原有的事件也可以被触发

2.4 position:absolute+left负值

设置position:absolute绝对定位，然后设置一个特别大的left负值，使元素定位在可见区域外

2.5 transform:scale(0)

将一个元素设置为缩放无限小，元素将不可见，元素原来的位置将被保留

3 rgba()和opacity透明效果

rgba()和opacity都能实现透明效果，不同之处在于：

opacity作用于元素，以及元素内的所有内容的透明度，即所有子元素都会继承opacity

rgba()只作用于该元素的颜色和背景色，子元素不会继承rgba()的效果
