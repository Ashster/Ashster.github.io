CSS布局（一）盒模型

发表于2021-05-28|更新于2021-05-31|CSS
|阅读量:

1 盒模型

盒模型由 content+padding+border+margin组成

1.1 标准盒模型

标准盒模型中，width和height指的是content的宽和高

1.2 怪异盒模型（IE盒子模型）

怪异盒子模型中，width和height指的是content+padding+border部分总体的宽和高

1.3 box-sizing属性（css3）

box-sizing属性有以下指：

  • content-box 也就是标准盒子模型，width和height只包含content
  • padding-box width和height包含content+padding
  • border-box 怪异盒子模型，width和height包含content+padding+border

2 盒模型相关标签

标签大致分为三类，分别是：块级元素、行级元素、行内块元素

2.1 块级元素

div,p,h1-h6,ol,ul

以及设置display:block转换成块级元素的其他

块级元素特点：

  1. 一个块级元素独占一行
  2. 元素width height line-height margin-top margin-bottom均可以设置
  3. 元素width不设置的情况下，默认为父级容器的100%也就是和父级容器宽度一致

2.2 行级元素

a,span,strong 等

以及设置display为inline转换成行级元素的

行内元素特点：

  1. 一行内可以有多个行内元素
  2. 元素width height无效，margin上下放下无效，左右方向有效，padding上下左右都有效
  3. 元素宽度就是它包含的内容本身的宽度，不可改变

2.3 行内块元素

img,input等

以及设置display为inline-block转换成行内块元素的

行内块元素特点：

  1. 一行内可以有多个行内块元素（inline特点）
  2. 元素的width height line-height margin-top/bottom 均可以设置（block特点）

3 盒模型属性值

3.1 width和height

任何块级元素都可以设置width和height

width：auto 尽可能地宽

height：auto 尽可能地窄

3.2 内边距padding

padding:50%可以实现正方形效果，因为水平和垂直padding的百分比都是相对于包含块的宽度决定

3.3 外边距margin

外边距margin会在元素创建额外的空白

3.3.1 margin重叠/margin合并

外边距合并是指当两个垂直外边距相遇时，他们将形成一个外边距。

合并的外边距的高度等于发生外边距合并时两个外边距的高度中的较大者。

只有普通文档流中的块级元素的垂直外边距才会发生外边距合并。

行内块、浮动块或者绝对定位之间的外边距不会发生合并。

发生合并的情况：

  1. 两个有垂直外边距的块级元素 是 兄弟关系

  2. 两个元素是父子关系 父亲没有内边距孩子没有边框把margin分隔开

  3. 一个空元素 ，没有边框和填充时，外边距会合并成为一个外边距

    这时如果外边距遇到另一个元素的外边距，还能继续发生合并

3.3.2 BFC——margin塌陷解决方案

BFC（Block Formatting Contexts）块级格式化上下文，属于普通流

具有BFC特性的元素可以看作是隔离了的独立容器，容器里面的元素不会在布局上影响外面的元素。

触发BFC的条件：

  • body根元素
  • 浮动元素，float：left/right 除none以外的之
  • 绝对定位元素 position：absolute/fixed
  • display：inline-block/table-cells/flex
  • overflow：除了visiable以外的值，也就是hidden/auto/scroll

3.4 border

圆形：元素宽高相同，border-radius：50%

半圆：元素宽高不同，且圆角半径要和宽高配合

扇形：元素宽高及一个圆角半径相同
