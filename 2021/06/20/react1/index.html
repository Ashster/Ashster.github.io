1.JSX简介

JSX: JavaScript的语法扩展

JSX可以生成React元素,React中配合使用JSX可以更好的描述UI应该呈现出它应有交互的本质
JSX看起来可能有些类似模板语言，但是他拥有JS的所有功能

1.1.1 为什么使用JSX？

这要从React的逻辑来看，
React认为渲染逻辑本质上与其他UI逻辑内在耦合，React并没有将标记与逻辑强行分离，
而是将标记和逻辑共同存放在称为“组件”的松散耦合单元。
React并不是强制要求使用JSX，但是使用JSX会更加的方便，视觉上也有辅助作用

1.1.2 在JSX中嵌入表达式

我们上面已经说过了，JSX拥有JS的所有功能
在JSX中，你可以通过一对大括号{}，在大括号内放置任意js表达式

1.1.3 JSX也是一个表达式

在编译之后，JSX表达式会被转换成普通的JavaScript函数调用，并且对其取值后得到JavaScript对象

也就是说，可以在if或者for循环的代码块中使用JSX

  • 将JSX赋值给变量
  • 将JSX作为参数传入
  • 从函数返回JSX

1.1.4

JSX设置特定属性的值有两种方式

  • 使用引号，将属性值指定为字符串字面量
  • 使用大括号，在属性值中插入一个JS表达式

注意，对于同一个属性值，两种方式只能使用一个。

React使用小驼峰命名法来定义属性名

2. 元素渲染

元素：元素是构成React的最小单元，元素描述了在屏幕上显示的内容

与浏览器DOM元素不同，React元素是创建开销极小的普通对象。
React DOM会负责更新DOM来与React元素保持一致

2.2.1 将一个元素渲染为DOM

采用ReactDOM.render进行将react 元素渲染到指定位置
一般都是渲染到根节点

2.2.2 更新已渲染的元素

React元素一旦被创建，就无法改变他的子元素或者属性，是不可变对象
每个元素就像是电影的一个单帧，他只代表了某个时刻的UI情况

目前，更新UI的唯一方式就是创建一个全新的元素，然后将他传入ReactDOM.render
也就是说，更新一次就要调用一次ReactDOM.render

后续我们会学习只调用一次ReactDOM.render的方法，把代码封装到有状态组件中去

2.2.3 React 只更新它需要更新的部分

React DOM 会将元素和它的子元素与它们之前的状态进行比较，并只会进行必要的更新来使 DOM 达到预期的状态。
