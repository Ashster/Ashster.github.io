1. 组件&Props

组件允许将UI拆分为独立可复用的代码片段，并对每个片段进行独立构思。
利用组件的思想，可以把UI拆分成一个个独立可复用的代码片段。
组件类似于JavaScript中函数的概念，它接收任意的参数props，并返回用于描述页面内容的React元素,注意，组件名称必须以大写字母开头

组件有两种：

  • 函数组件

  • class组件

1.1 自定义组件&渲染组件

当React元素为用户自定义组件时，它会将JSX所接收的属性以及子组件转换为单个对象传递给组件，这个对象称之为props

渲染过程分析：

  1. 首先调用ReactDOM.render()函数，传入element的值作为参数
  2. React调用Welcome组件，将参数对象props也就是{name:’aaa’}作为props传入
  3. Welcome组件将

    Hello,aaa

    元素作为返回值返回
  4. React DOM将DOM高效更新为

    Hello,aaa

1.2 组合组件

组件可以在其输出中引用其他的组件。这样可以让我们在同意组件中抽象出任意层次的细节。

1.3 提取组件

我们可以将组件拆分为更小的组件，然后层层套叠使用

1.3 props的只读性

所有React组件都必须像纯函数一样保护他们传入的props不被更改。

组件无论是使用函数声明还是通过class声明，都不能修改自身的props属性。

2. State&生命周期

在不使用状态state之前，我们想要更新ui，就必须通过ReactDOM.render进行重新渲染。

state可以让组件实现自动更新
state和props类似，但是state是私有的，完全受控于当前组件

class组件才具有state，所以我们想使用state就必须先把函数组件转换成class组件（后续react hooks会解决这个问题）

当组件更新时render方法被调用，但是只要在相同的DOM节点中渲染相同的class组件，就只有一个该class组件的实例被创建使用，这就可以让我们使用如state或者生命周期方法等很多其他特征。

2.1 将函数组件转换为class组件

  1. 创建一个同名的ES6 class，且继承于React.Component
  2. 添加一个空的render()方法
  3. 将函数体移动到render()方法中去
  4. 在render()方法中使用this.props替换props
  5. 删除剩余的空函数声明

2.2 在class组件中添加局部state

  1. 把render()方法中的props替换成state
  2. 添加一个class构造函数constructor，在该函数中为this.state赋初值
  3. 移除元素原来的属性

2.3 将生命周期添加到class组件当中

当React组件第一次渲染到DOM中时，称为“挂载mount”
当React组件在DOM中被删除时，称为“卸载unmount”

生命周期方法：在class组件中声明一些特殊的方法，当组件挂载或者卸载时会执行这些方法

  • componentDidMount()该方法会在组件已经渲染到DOM中后运行
  • componentWillUnmount()该方法在组件从DOM中卸载时运行

2.4 State正确使用规则

  • 不要直接修改state，而是要使用setState()去修改state

  • state的更新可能是异步的，处于性能考虑，React可能会把多个setState()合并成一个去调用，所以this.props和this.state可能会异步更新，所以，不要依赖他们本身的值来更新下一个状态

    异步更新问题的解决方案：可以让setState()接收一个函数而不是一个对象，这个函数用上一个state作为第一个参数，将本次更新被应用时的props作为第二个参数：

  • State的更新会被合并

    当调用setState()时，React会把你提供的对象合并到当前state中去，这里的合并是浅合并

    也就是说，你修改部分就合并一部分，其余的保持不变

  • 数据是向下流动的

    state除了拥有它的组件，其他组件都无法访问，但组件可以把它的state作为props传递到他的子组件中，子组件会在props中接收state的相关参数，但是组件本身并不知道这个参数是来自父组件的state还是props或是手动输入。

    这就是所谓的自上而下/单向的数据流
    任何的state总是所属于特定的组件，而且从该state派生的任何数据或UI只能影响低于他们的组件
