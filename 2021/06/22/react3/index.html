1. 事件处理

React元素的事件处理和DOM元素的事件处理类似，但有一些不同：

  • React元素的事件命名采用小驼峰式

  • 使用JSX语法时需要传入一个函数作为事件处理函数{}，而不是传入一个字符串””

  • 在React必须显示使用preventDefault来组织默认行为，不能通过返回false的方式组织默认行为
    使用React时，一般不需要使用addEventListener为已创建的DOM元素添加监听器，只需要在该元素初始渲染的时候添加监听器即可。

  • 使用ES6 class定义一个组件的时候，通常做法是将事件处理函数声明为class中的方法
    js中，class的方法默认不会绑定this，所以要绑定this，采用bind

    例如 this.handleClick=this.handleClick.bind(this);

    或者使用实验性的public class fields语法，绑定回调函数

    handleClick=()=>{}

    如果你没有使用class fields语法，你可以在回调中使用箭头函数

    <button onClick={()=>this.handleClick()}

1.1 向事件处理程序传递参数

传递参数，两种方式实现

  • 箭头函数
  • Function.prototype.bind

在这两种情况下，事件对象e会被作为第二个参数传递，如果是通过箭头函数，事件对象必须显示进行传递，如果通过bind方式，事件对象以及更多的参数会被隐式传递。

React中，可以创建很多不同的组件来封装所需行为。然后，根据不同的状态，只渲染对应状态下的部分内容。

使用if运算符或者条件运算符去创建元素表示当前的状态，然后让React根据他们更新UI。

3.3 Key

当我们创建元素列表时，必须为每个元素分配一个独一无二的key
key值帮助React识别哪些元素改变了，比如被添加或者删除。有了这个独一无二的key，才可以精准捕捉到这个元素的变化。

一般是使用数组中的id作为元素的key，尽量不要用元素索引index作为key

索引作为key会有很多问题：
如果列表项目的顺序可能变化，就不建议直接使用索引来作为key，这样做会导致性能拜尼察，还可能引起组件状态问题。如果你不显示的指定key值，React将默认使用index索引作为列表项目key值
