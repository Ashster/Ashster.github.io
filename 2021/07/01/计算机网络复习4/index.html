第四章 网络层

1. 网络层提供的两种服务

网络层提供服务的特点：网络层向上提供简单的、无连接的、尽最大努力交付的数据报服务，不保证通信可靠性。

网络层提供的两种服务：

  • 虚电路服务：保证可靠通信，必须建立连接
  • 数据报服务：尽最大努力发送，不保证可靠通信
    • 思路：可靠通信由用户主机保证
    • 连接：不需要建立连接
    • 终点地址：每个分组都有完整重点地址
    • 分组转发：每个分组独立选的路由器进行转发
    • 当结点故障时：出故障的结点可能会丢失分组，一些路由可能会发生变化
    • 分组的顺序：到达顺序不一定是发送顺序
    • 端到端的差错控制：由用户主机负责

2.网际协议IP

与网际协议IP配套使用的三个协议：

  • 地址解析协议ARP
  • 网际控制报文协议ICMP
  • 网际组管理协议IGMP

2.1 虚拟互联网络

虚拟互联网络：逻辑互联的网络，可以由多种异构网络互联而成，在网络成看起来像一个统一的网络

中间设备：用来将网络互联

这些中间设备的总结：

  • 中继器（也叫做转发器）：物理层，用于在两个网络节点之间物理信号的双向转发。中继器的作用就是将信号放大，因为信号在传输过程中会衰减，中继器能使信号走得更远。
  • 集线器：物理层，是有多个端口的特殊的中继器（转发器），集线器可以实现多台计算机之间的互联，因为它有很多的端口，每个口都能连计算机。
  • 网桥：数据链路层，将两个局域网LAN连接起来，根据MAC地址来转发帧
  • 交换机：数据链路层，可以理解为网桥高配版，有网桥的功能但是比网桥强。交换机常常用来连接独立的计算机，端口比网桥多，而网桥链接的目标是局域网LAN。
  • 路由器：网络层，路由器为经过路由器的每个IP数据包寻找一条最佳的传输路径，并将该数据有效地传送至目的站点。路由器的基本功能是:把数据（IP报文）传送到正确的网络。
  • 网关（网间连接器，协议转换器）：在网络层以上 实现网络互连，仅用于两个高层协议不同的网络互联。网关与网桥不同的是，网关要对收到的信息重新打包，以适应目的系统的需求。网管就是网络的关口。

交换机与路由器的区别：

  • 工作层次不同：交换机是数据链路层的设备，路由器是网络层的设备。
  • 转发依据不同：交换机转发所依据的对象是MAC地址（物理地址）
                           路由器转发所依据的对象是IP地址（网络地址）
  • 主要功能不同：交换机主要用于组建局域网;路由器主要功能是将交换机组好的局域网相互连接起来，或者接入Internet 。交换机能做的路由器都可以做。交换机不能分割广播域，路由器可以。路由器配置比交换机更复杂。
  • 总结：路由器实现了不同网络之间的数据转发，交换机实现了特定网络内的数据交换

2.2 分类的IP地址

2.2.1 IP编址的三个历史阶段：
  • 分类的IP地址：最基本的编址方式
  • 子网的划分：对基本编址方式的改进
  • 构成超网：比较新的无分类编址方式
2.2.2 分类的IP地址

分类的IP地址可以记为：

​ IP地址::={<网络号>,<主机号>}，都是32位，4个字节

A、B、C类地址都是单播地址，D类地址用于多播

  • A类：最大网络可指派数126（2^7-2），8位中1位位固定位，剩7位，减2是因为要出去全0位和全1位。可用网络号范围：1——126。

  • B类：最大网络可指派数16383（2^14-1）,16位中固定两位，剩14位，因为固定位为10，所以不存在全0或全1的情况，但是规定128.0.0.0不使用，所以-1。范围 128.1——191.255

  • C类：最大可指派网络数2097151（2^21-1）,24位中固定三位，剩21位，因为固定位为110，所以不存在全0或者全1的情况，-1原因同B类。范围 192.0.1——223.255.255

一般不使用的特殊IP地址：

  • 0.0.0.0 表示这个（this）
  • 网络号为127 表示环回测试
  • 主机号全0 表示本主机连接到的单个网络地址
  • 主机号全1 网络上的所有主机
  • 不指派的网络 128.0.0.0 192.0.0.0
  • 只能用于机构内部的通信的本地地址：10.0.0.0-10.255.255.255，172.16.0.0-172.31.255.255，192.168.0.0-192.168.255.255

IP地址重要特征：

  • 每个IP地址都是由网络号和主机号两部分构成，是一种分等级的地质结构
  • 实际上IP地址是表只一个主机或路由器和一条链路的接口
  • 用转发器或者网桥连接起来的若干局域网仍是一个网络，这些局域网具有相同的网络号
  • 所有分配到网络号的网络都是平等的

网络上的IP地址：

  • 同一个局域网的各个IP地址的网络号相同
  • 用网桥互联的网段仍是一个局域网，只有一个网络号
  • 一个路由器有多个接口，每个接口对应不同的网络号
  • 两路由器直接相连时，可以不给两端接口分配IP，这样的特殊网络叫做无编号/无名网络
2.2.3 IP地址与硬件地址

IP地址与硬件地址的区别：

  • 层次上：
    • 硬件地址或物理地址是在数据链路层和物理层使用的地址
    • IP地址是在网络层及以上各层使用的地址，是一种逻辑地址

2.2.4 地址解析协议ARP

​ 地址解析协议ARP实现从网络层的IP地址到数据链路层的硬件地址MAC地址的转换。因为在传播过程中，源IP地址目的IP地址始终不变，但是源MAC地址和目的MAC地址在每条链路上都要变化。
​ 每个主机内部都有一个ARP高速缓存（ARP cache），里面有所在局域网上的各主机和路由器的IP地址到硬件MAC地址的映射表。因为主机的硬件地址可能会发生改变，所以ARP高速缓存中的映射表也会经常更新，映射表中的每个项目都有生存时间，超过生存时间的项目会被删掉。

工作流程：当主机A向该局域网上某个主机B发送IP数据报时，先在ARP高速缓存中查看有没有主机B的IP地址，如果有，可以直接查出目的IP对应的MAC地址，如果没有，就执行ARP请求分组：

1. ARP请求分组：在局域网广播一个ARP请求分组，包含源硬件MAC地址，源IP地址，目的MAC地址（未知时为0）, 目的IP地址。
 2. 本地广播ARP请求，路由器不转发ARP请求
 3. ARP响应分组：某主机收到广播，发现本机IP与查询IP一致，就回复ARP相应分组，包含发送方MAC地址，发送方IP地址；同时，将请求分组中IP地址与MAC地址的对应关系保存
 4. 收到回复的ARP响应后，将对应IP和MAC地址存入ARP高速缓存中，方便下次使用

生存时间：ARP高速缓存中的每条映射都只存在一段时间，超时后会被删除

ARP地址解析协议 特点：

ARP协议只能解决同一局域网上IP地址和硬件地址的映射问题，不在同一局域网则无法使用，ARP工作过程对于用户来说是透明的。

四种使用ARP的典型情况：

(1)发送方是主机，要把IP数据报发送到本网络上的一个主机，这时ARP找到目的主机的硬件地址

(2)发送方是主机，要把IP数据报发送到另一个网络上的一个主机，这时ARP找到本网络上的一个路由器的硬件地址，剩下的工作由这个路由器完成

(3)发送方是路由器，要把IP数据报发送到本网络上的一个主机，这时ARP找到目的主机的硬件地址

(4)发送方是路由器，要把IP数据报发送到另一个网络上的主机，这时ARP找到本网络上的一个路由器的硬件地址，剩下的工作由这个路由器完成

2.2.5 IP数据报格式

IP数据包=首部+数据部分

首部=固定部分（20字节）+可变部分

各字段含义：

  • 版本：4位，IP协议版本，IPv4或IPv6，通信双方使用的IP协议版本必须一致
  • 首部长度：4位，因为首部存在可变部分，所以长度20-60字节，且以4字节为单位，必须是4的整数倍，不是要填充至整数倍
  • 区分服务：8位，区分服务时才有用，一般不用
  • 总长度：16位，首部+数据的总长度，单位字节，IP数据报最大长度为2^16-1即65535字节
  • 标识：16位，同一个数据报的不同分片标识相同。因此接收方能根据标识将不同分片重装。
  • 标志：3位，最低位1标识后面还有分片，最低为0标识这是最后一个分片。中间位1标识不能分片，中间位0标识可以分片。首位无含义。
  • 片偏移：13位，表示较长的分组分片后，某片在原分组的相对位置，单位是8字节，所以每个分片的长度都是8字节的整数倍。
  • 生存时间：8位，表明数据报在网络中的寿命，单位是跳数，表明数据报在互联网中至多可经过多少个路由器。
  • 协议：8位，表明使用哪种协议
  • 首部检验和：16位，这个字段只能检验首部，不能检验数据部分。数据报每经过一个路由器，路由器就要重新计算一下首部检验。
  • 源地址：32位，8字节
  • 目的地址：32位，8字节
  • 可变部分：长度1-40字节，IPv4中可能有用，IPv6取消了可变部分

3. 划分子网和构造超网

3.1 划分子网

两级IP地址（网络号+主机号）缺点：IP地址空间利用率有时很低，给每个物理网络分配一个网络号会使路由表变得很大，网络性能变化，两级IP地址不够灵活

划分子网的基本思路：借用IP地址中主机号的若干位作为子网地址

划分后的IP地址：|网络号|（子网）|主机号|

子网掩码

  • 作用：用来找出IP地址中的子网部分，长度位32位，原IP中网络号和子网部分置1，主机号部分置0
  • 默认子网言吗：在不进行子网划分的时候，也要给出子网掩码，就是默认子网掩码

子网掩码是一个网络或者一个子网的重要属性，路由表中的每一项还要给出该网络的子网掩码。

子网数：2^k-2,k表示子网号的位数，去掉全0全1，就是可用子网数

子网划分方式：

1.  固定长度划分：所划分的子网的子网掩码都相同，取所用最大的主机数进行划分呢
 2.  变长划分：根据需求灵活分配
3.2 使用子网的分组转发💙💙💙💙：
子网划分后的路由表：包含目的网络地址、子网掩码、下一跳地址

子网划分后的路由器分组转发算法：

        1. 从数据报的首部提取出目的IP地址 D
           2. 判断是否为直接交付。对路由器直接相连的网络逐个检查：用各网络的子网掩码逐个与D按位相与，看结果是否和相应的网络地址匹配。如果匹配，就直接交付；否则就是间接交付，继续执行。
           3. 若路由表中有目的地址为D的特定主机路由，则把数据报传送给路由表中所指明的下一跳路由。否则继续执行。
           4. 对路由表中的每一行，用其中的子网掩码与D逐位相与，若结果与该行的目的网络地址匹配，则把数据报传送给该行所指明的下一跳路由器，否则继续执行。
           5. 若路由表中有一个默认路由，则把数据报传送给路由表所指明的默认路由器，否则继续执行。
           6. 报告转发分组出错。

4. 网际控制协议ICMP

ICMP允许主机或路由器报告差错和提供有关异常情况的报告。

4.1 ICMP报文种类：差错报告报文，询问报文

4.2.ICMP的应用：

  • PING： PING即Packet InterNet Groper，用于探测两台主机间是否连通，源主机向目标主机发送ICMP的回送请求报文（封装在IP数据报中），目标主机若接收到该报文则返回回送回答报文
  • 路由探测： 路由探测即源主机向目标主机发送无法交付的UDP数据报（封装于IP数据报，若目标主机接收到该数据报，则会返回ICMP终点不可达报文），第一次发送时将IP数据报的生存时间设为1，这样一来第一个路由器接收到后将生存时间-1就会直接判断该IP分组需要丢弃，并返回ICMP时间超过报文，源主机接下来发送第二个IP数据报（依然为不可交付UDP数据报），此次将生存时间设为2……以此类推，直至接收到ICMP终点不可达报文，或生存时间达到上限为止。

5.互联网的路由选择协议

分层次的路由选择协议：互联网采用分层次的路由选择协议，自适应的（动态的）、分布式路由选择协议

自治系统AS：

  • 在单一技术管理下的一组路由器，在AS内部使用内部网关协议，AS之间使用外部网关协议
  • 内部网关协议IGP：在一个自治系统内部使用，RIP,OSPF
  • 外部网关协议EGP：在不同自治系统之间使用，如BGP

5.1 内部网关协议RIP

  • 是一种分布式，基于距离的路由选择协议

  • 距离：直连网路距离为1，每通过一个非直连网络距离增加1，距离也称作跳数，每经过一个路由器跳数就增加1，距离实际上是指最短距离

  • RIP允许一个路径最多包含15个路由器，也就是距离最大值为16，故RIP适合小型互联网使用；RIP不能在两个网络之间同时使用多条路由

  • 工作流程：每个路由器每隔一段时间向外广播，每个路由器收到广播后更新自己的路由表

  • 刚开始时只知道直连网络的距离，路由表为空，以后，每个路由器只和数目有限的相邻路由器交互并更新路由信息，经过若干次更新后，所有路由器最终会知道到达本自治系统其他路由器的最短距离和下一跳地址，此时称该网络收敛

  • RIP协议的特点：

    (1)仅和相邻路由器交换信息，不相邻的路由器不交换信息

    (2)交换的信息是当前本路由器所知道的全部信息，即其现在的路由表

    (3)按固定时间间隔交换信息

  • 距离向量算法：

    • 路由器收到地址为X的相邻路由器的一个RIP报文，修改报文中所有项目，把下一跳地址改为X，把距离字段值都+1
    • 对修改后的RIP报文中的每个项目重复以下步骤：
      • 若项目中目的地址不在路由表中，则加入路由表
      • 若下一跳给出的路由器地址与RIP报文传来地址相同，则用收到的项目替换原路由表中的项目
      • 若收到项目中的距离小于路由表中的距离，进行更新
    • 若超过3分钟未收到相邻路由信息，则将其标记为不可达，即把距离置为16
