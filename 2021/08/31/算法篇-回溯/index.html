回溯法

回溯法回溯算法的基本思想是：从一条路往前走，能进则进，不能进则退回来，换一条路再试。

回溯算法实际上一个类似枚举的搜索尝试过程，主要是在搜索尝试过程中寻找问题的解，当发现已不满足求解条件时，就 “回溯” 返回，尝试别的路径。

回溯算法的基本思想是：从一条路往前走，能进则进，不能进则退回来，换一条路再试。回溯和深度优先搜索DFS很像，都要遍历所有答案集合，但是回溯更强调回退这个过程。

第一类：排列组合子集问题

思考他们的剪枝条件，有的时候用 used 数组，有的时候设置搜索起点 begin 变量，**

46.全排列（不含重复数字）

https://leetcode-cn.com/problems/permutations/

剪枝条件是不可重复选，所以用一个used对象来表示已经选过的数字

47.全排列II（包含重复数字）

https://leetcode-cn.com/problems/permutations-ii/

这个相对一上一个来说，剪枝条件更为复杂，先对传入的nums数组按照从小到大排序方便剪枝：

  • 已经选过的数字，剪掉
  • 同一层和前一个数字相同且前一个数字没有被使用，为了避免重复，直接剪掉这个整支

77.组合

https://leetcode-cn.com/problems/combinations/

给定两个整数 n 和 k，返回范围 [1, n] 中所有可能的 k 个数的组合。

剪枝条件：每次递归中控制枚举的范围，下一层选择的范围起点是这一次选择的位置+1

39.组合总数

https://leetcode-cn.com/problems/combination-sum/solution/

给你一个数组，里面都是不带重复的正数，再给定 target，求出所有和为 target 的组合。
元素可以重复使用，但组合不能重复，比如 [2, 2, 3] 与 [2, 3, 2] 是重复的。

剪枝条件：

  • 和组合数类似，也是不向前寻找要向后寻找，但是区别是，这里的元素可以重复使用，所以我们向后寻找要从上次使用的元素位置开始，包括上次使用的元素位置。

多一层结束条件：

  • 大于等于目标和就结束，但只有等于目标和才会放入结果中

40.组合总数II

https://leetcode-cn.com/problems/combination-sum-ii/solution/

和39.组合总数的区别就是，元素不可以重复使用。一样的是，组合不可以重复。

所以我们要在39的基础上多一层剪枝再做一些修改：

  • if (i - 1 >= start && candidates[i - 1] == candidates[i]) 当前选项和左邻选项一样，跳过，剪枝
  • backtrack回溯采用i+1而不是39中的i了，因为元素不能重复使用，必须从当前使用元素的后面一个（不包括当前使用元素开始）

78.子集

https://leetcode-cn.com/problems/subsets/solution/

对于每个元素，都有两条路：加入path，不加入path。选这两种的任意一个，都可以进入到下一个元素的选择中去。

  • 用索引index代表当前递归考察的数字nums[index]。
  • 当index越界时，说明所有数字考察完了，得到一个解，把它加入解集，结束当前递归分支。

90.子集II（包含重复元素）

https://leetcode-cn.com/problems/subsets-ii/

和上一题换一种思路来解，因为是查找所有不重复的子集，所以temp集合每次都先把自己当前的结果放到res中作为子集的一个解，之后再向里面添加新的元素。

因为数组中包含重复元素，首先要通过sort（）排序让相同的元素紧挨着，方便我们剪枝：

  • 对于同一层的元素，如果他的前一个和他一样，就直接把这个支减去
